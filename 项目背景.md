# 会议纪要 App 项目背景

## 项目概述

会议纪要项目的移动应用版本。已从 Tauri + React + Rust 重写为 Flutter。

新项目位于：`../nivo_app_flutter`

## 技术栈（当前）

- Flutter 3.29+, Dart 3.7+
- Provider 状态管理
- Supabase Auth 认证
- dio HTTP 客户端
- sherpa_onnx 端侧语音识别
- record 跨平台录音

## 核心功能

### 1. 登录
- Supabase 邮箱登录
- 超级管理员权限检查

### 2. 会议录制
- 实时语音转录（云端 SSE / 端侧 sherpa_onnx）
- 行业选择（企业服务、金融、消费、科技、制造）
- 模板选择（经典模式/场景模式/自定义模式）
- AI 会议纪要生成

### 3. 历史记录
- 分页列表 + 时间筛选
- 查看详情、修改标题、删除

### 4. 设置
- ASR 模式切换（云端/本地）
- 端侧模型下载管理
- 云端 API 配置

## 后端接口

对接 meetAgHK（Java + Spring Boot），基础地址：`https://www.nivowork.cn`

- `POST /auth/checkSuperAdmin` — 权限检查
- `GET /api/speech/start` — SSE 实时转录
- `POST /api/speech/audio` — 发送音频数据
- `POST /api/speech/stop` — 停止转录
- `POST /api/chat/run` — 生成会议纪要
- `GET /db/getHistoryList` — 历史列表
- `POST /db/delHistory` — 删除历史
- `POST /db/updateHistoryTitle` — 更新标题

## 开发注意事项

- 内测阶段，使用预填测试账号
- 端侧 ASR 需要先下载模型（设置页操作）
- 后端设计围绕局部页面展开
