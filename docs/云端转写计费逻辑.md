# 云端转写计费逻辑

## 三层时长架构

| 层级 | 说明 | 生命周期 | 存储 |
|------|------|---------|------|
| 全局时长 | 本月累计云端转写秒数 | 跨会议 | 远端 |
| 会议时长 | 本次会议累计秒数 | 单次会议 | 内存 |
| 段时长 | 暂停→恢复之间的秒数 | 单段录音 | 内存 |

## 数据流

```
开始会议（全局收音 ON）
  │
  ├─ fetchConfig(userId) → GET /api/recording-duration/config
  │   返回 { usage: 已用秒数, limit: 月限额 }
  │
  ├─ 检查 isLimitReached → 超限则阻止开始
  │
  └─ startSegment()
       │
       ├─ 每秒：segmentDuration++, meetingDuration++, globalUsage++
       │
       └─ 每60秒：reportRecordingDuration(userId, segmentDuration)
            POST /api/recording-duration/report?userId=xxx&duration=xxx
            上报后 segmentDuration 归零

暂停
  └─ stopSegment()
       └─ 上报剩余 segmentDuration → segmentDuration 归零

恢复
  └─ startSegment()（新段从0开始）

结束会议
  └─ endMeeting()
       └─ stopSegment()（上报剩余）→ meetingDuration 归零
```

## API

### 获取配置
```
GET /api/recording-duration/config?userId={userId}
Response: { code: 200, data: { usage: 3600, limit: 18000 } }
```

### 上报时长
```
POST /api/recording-duration/report?userId={userId}&duration={seconds}
```

## 超限处理

- 开始会议前检查 `globalUsage >= globalLimit`
- 超限时提示用户升级套餐，不启动云端转写
