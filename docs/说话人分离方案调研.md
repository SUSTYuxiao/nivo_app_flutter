# 说话人分离方案调研

## 当前状态

### Flutter App（已实现）
- AfterMeet：云端 OSS + `/a2t/processV2`，有说话人分离 + 时间戳；iOS 可选本地 SFSpeechRecognizer（无说话人分离）
- InMeet 实时：SSE 云端转写 / iOS SFSpeechRecognizer / Android sherpa_onnx，均无说话人分离
- InMeet 结束时：上传录音走 processV2，有说话人分离；失败时 fallback 到实时转录文本

### meetAg Web
- AfterMeet：云端 `/a2t/processV2`，有说话人分离 + 时间戳
- InMeet 实时：SSE 转写，无说话人分离（全部标记"发言人"）
- InMeet 结束时"高精度转写"：上传录音走 `/a2t/processV2`，有说话人分离

## 云端方案（meetAg 现有流程）

### 流程

```
音频文件
  ↓
预处理（多声道 → mono WAV）
  ↓
GET /oss/getStsToken → 获取阿里 OSS 临时凭证
  ↓
multipartUpload → OSS "audio/{filename}"
  ↓
GET /a2t/processV2?filePath=audio/{filename}
  ↓
返回 TranscriptionSentence[]
  ↓
格式化: "**发言人1 - 2分30秒**\n\n内容"
  ↓
喂给 chatRun / chat/sse 生成纪要
```

### API

| 端点 | 方法 | 用途 |
|------|------|------|
| `/oss/getStsToken` | GET | 获取 OSS STS 临时凭证（region, bucket, accessKeyId, accessKeySecret, securityToken） |
| `/a2t/processV2` | GET | 离线转写 + 说话人分离，参数 `filePath` |
| `/api/transcription/merge` | POST | InMeet 实时合并转录段（无说话人分离） |

### processV2 响应格式

```json
{
  "success": true,
  "data": [
    {
      "beginTime": 150000,
      "endTime": 155000,
      "text": "这是第一段发言",
      "sentenceId": 1,
      "speakerId": 0,
      "speakerName": "发言人1"
    }
  ]
}
```

### OSS 配置
- 凭证通过 STS 动态获取，不硬编码
- 上传路径：`audio/{filename}`
- 使用 multipart upload

### 音频预处理
- 多声道 → 混合为 mono
- InMeet 录音已是 16kHz mono WAV，跳过预处理
- 预处理失败时静默回退到原始文件

## 本地方案：FluidAudio

### 概述
- GitHub: https://github.com/FluidInference/FluidAudio
- 开源 Swift SDK，Apache 2.0
- 完全端侧推理，使用 CoreML + Apple Neural Engine

### 能力

| 功能 | 支持 | 详情 |
|------|------|------|
| ASR | 是 | Parakeet TDT 模型，支持中文（v3 多语言版） |
| 说话人分离 | 是 | 离线 ~13% DER，实时 ~26% DER |
| VAD | 是 | 语音活动检测 |
| TTS | 是 | 文本转语音 |

### 平台限制
- iOS 17.0+ / macOS 14.0+ only
- 无 Android 支持，无计划
- 无官方 Flutter 插件，需自建 Platform Channel

### 模型大小
- 说话人分离：~100 MB（pyannote segmentation + WeSpeaker embeddings）
- ASR：110M-600M 参数，视精度而定

### 性能
- M1 上 RTF 0.017（60x 实时速度）
- 60 分钟音频 < 30 秒处理

### Flutter 集成方式
与现有 `NativeAsrPlugin.swift` 模式一致：
```swift
import FluidAudio
// MethodChannel 接收音频数据
// FluidAudio 处理 ASR + 说话人分离
// EventChannel 返回结果
```

### 对比

| | FluidAudio | sherpa-onnx | 云端 /a2t/processV2 |
|---|---|---|---|
| 平台 | iOS only | iOS + Android | 全平台 |
| 说话人分离 | 是 | 否 | 是 |
| 离线 | 是 | 是 | 否 |
| 中文 ASR | 是（v3） | 是 | 是 |
| 模型下载 | ~100-600MB | ~50-200MB | 无需 |

## 实施路径

### 短期（当前）
Flutter 接入 meetAg 云端流程：
1. AfterMeet：OSS 上传 + `/a2t/processV2` → 说话人分离
2. InMeet 结束时：上传录音走 processV2 → 说话人分离
3. 两端统一，与 meetAg 行为一致

### 中期
iOS 集成 FluidAudio：
- 本地 ASR + 说话人分离，减少云端依赖
- 作为离线备选方案
- 需要 iOS 17+ 最低部署目标

### 长期
- Android 说话人分离：等待 sherpa-onnx 或其他方案支持
- 或 Android 始终走云端
